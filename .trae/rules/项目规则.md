# MomentFlow 项目规则

## 1. 技术栈

### 1.1 前端 (Android)
- **开发语言**: Kotlin
- **最低 SDK 版本**: API 24 (Android 7.0)
- **目标 SDK 版本**: API 34 (Android 14)
- **架构**: MVVM + Clean Architecture
- **UI 框架**: Jetpack Compose
- **视频播放**: ExoPlayer
- **网络请求**: Retrofit + OkHttp
- **依赖注入**: Hilt
- **异步处理**: Kotlin Coroutines + Flow
- **JSON 序列化**: kotlinx.serialization

### 1.2 主业务服务 (Backend)
- **框架**: NestJS (Node.js)
- **ORM**: Prisma
- **数据库**: PostgreSQL
- **缓存**: Redis (ioredis)
- **实时通信**: Socket.io (可选)

### 1.3 AI 分析服务 (AI Service)
- **框架**: FastAPI (Python)
- **Web 服务器**: Uvicorn
- **AI 框架**: PyTorch / HuggingFace Transformers
- **视频处理**: OpenCV + FFmpeg
- **音频处理**: Librosa

### 1.4 部署
- **容器化**: Docker + Docker Compose

## 3. 命名规范

### 3.1 文件与目录
- **Android 目录/文件**: camelCase 或 PascalCase
  - 示例: `DramaListScreen.kt`, `EmojiBurst.kt`
- **NestJS 模块**: kebab-case 目录，PascalCase 类名
  - 示例: `drama.module.ts`, `DramaService`
- **FastAPI 路由**: kebab-case
  - 示例: `analyze-video.py`

### 3.2 代码命名
- **变量/函数**: camelCase
- **常量**: UPPER_SNAKE_CASE
- **接口/类型**: PascalCase（TypeScript/Pydantic）
- **数据库表/字段**: snake_case

### 3.3 Git 提交信息
- 格式: `[类型] 简短描述
- 类型: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`
- 示例: `feat: add emotion heatmap component`

## 4. 开发规范

### 4.1 前端 (Android)
- 遵循 MVVM + Clean Architecture
- 使用 Jetpack Compose 构建 UI
- API 调用统一使用 Retrofit，封装在 `data/api/` 目录
- 数据模型统一放在 `data/model/` 目录
- ViewModel 统一放在 `ui/viewmodel/` 目录
- 使用 Hilt 进行依赖注入
- 使用 Kotlin Coroutines + Flow 处理异步任务
- 视频播放使用 ExoPlayer

### 4.2 后端 (NestJS)
- 遵循 NestJS 模块化架构
- 每个模块包含: `module.ts`, `controller.ts`, `service.ts`, `dto.ts`
- 使用 Prisma Schema 变更后记得生成迁移
- Redis 缓存优先使用

### 4.3 AI 服务 (FastAPI)
- 使用 Pydantic 做数据验证
- 视频分析任务使用异步处理
