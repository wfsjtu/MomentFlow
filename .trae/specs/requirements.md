# MomentFlow 项目需求说明书

## 一、功能需求

### FR-1：短剧列表页

**优先级：P0（必选）**

#### 场景：用户浏览短剧列表
- **WHEN** 用户进入应用首页
- **THEN** 系统展示短剧列表，每部短剧包含：封面图、标题、简介、时长、热度标签
- **AND** 列表支持瀑布流或网格布局，适配移动端竖屏

#### 场景：用户点击短剧进入播放
- **WHEN** 用户点击某部短剧卡片
- **THEN** 系统跳转至短剧播放页，自动开始播放

---

### FR-2：短剧播放器

**优先级：P0（必选）**

#### 场景：基础播放控制
- **WHEN** 用户在播放页
- **THEN** 系统提供：播放/暂停、进度条拖拽、音量控制、全屏切换
- **AND** 播放器支持 HLS/m3u8 流媒体播放

#### 场景：进度条情绪热力图
- **WHEN** 用户在进度条上悬停或长按
- **THEN** 系统展示该时间点附近的情绪热力分布（不同颜色代表不同情绪类型）
- **AND** 热力数据根据已观看用户的互动数据实时更新

---

### FR-3：高光点打标与下发

**优先级：P0（必选）**

#### 场景：高光点数据管理
- **WHEN** 运营人员（或 AI）为剧集标注高光点
- **THEN** 系统存储高光点信息：时间戳、高光类型（冲突/反转/名场面/撒糖）、互动组件类型、触发时长
- **AND** 支持批量导入高光点 JSON 数据

#### 场景：客户端接收高光点
- **WHEN** 用户播放短剧至某个时间点
- **THEN** 客户端根据预加载的高光点时间轴数据，提前 3 秒预加载互动组件资源
- **AND** 到达高光点时，自动弹出互动组件

#### API 设计

```
GET /api/v1/dramas/:id/highlights
Response:
{
  "drama_id": "xxx",
  "highlights": [
    {
      "timestamp": 45.2,          // 秒
      "type": "conflict",         // 高光类型
      "component": "emoji_burst", // 互动组件
      "duration": 5,              // 展示时长（秒）
      "emojis": ["🔥", "😱", "💥"], // 预设表情
      "label": "男主霸气反击"
    }
  ]
}
```

---

### FR-4：Emoji Burst 一键情绪互动

**优先级：P0（必选，核心功能）**

#### 场景：高光时刻触发互动组件
- **WHEN** 剧情到达高光时间点
- **THEN** 播放器底部/侧边弹出 Emoji Burst 组件，展示 3-5 个预设表情
- **AND** 组件以轻量动画呈现，不遮挡核心画面

#### 场景：用户点击表情表达情绪
- **WHEN** 用户点击某个预设表情（如 🔥）
- **THEN** 屏幕中央出现对应表情的粒子特效动画（从点击位置扩散/上升）
- **AND** 该表情的全局计数 +1，显示在组件旁

#### 场景：互动组件自动消失
- **WHEN** 高光窗口结束（默认 5 秒）
- **THEN** 互动组件以渐隐动画消失
- **AND** 如果用户未互动，不强制展示任何残留 UI

#### 场景：连续高光点处理
- **WHEN** 两个高光点时间间隔 < 3 秒
- **THEN** 系统合并为一个互动窗口，展示更丰富的表情选项

---

### FR-5：情绪热力图可视化

**优先级：P0（必选）**

#### 场景：情绪热力数据聚合
- **WHEN** 多个用户在同一时间段产生互动
- **THEN** 系统按时间窗口（每 5 秒）聚合互动数据
- **AND** 按表情类型分类统计计数

#### 场景：进度条热力渲染
- **WHEN** 用户在播放页查看进度条
- **THEN** 进度条上方/下方展示纵向热力条
- **AND** 不同颜色映射不同情绪类型（红色=激动、蓝色=感动、橙色=搞笑、紫色=震惊）

#### API 设计

```
GET /api/v1/dramas/:id/emotion-heatmap
Response:
{
  "drama_id": "xxx",
  "windows": [
    {
      "start": 0, "end": 5,
      "emotions": { "fire": 128, "shock": 45, "laugh": 32, "tear": 12 }
    },
    ...
  ]
}
```

---

### FR-6：互动数据上报与统计

**优先级：P0（必选）**

#### 场景：用户互动数据上报
- **WHEN** 用户点击互动表情
- **THEN** 客户端上报：用户标识（匿名 ID）、剧集 ID、高光点 ID、时间戳、表情类型
- **AND** 上报采用异步批量发送，不阻塞 UI 渲染

#### API 设计

```
POST /api/v1/interactions
Body:
{
  "drama_id": "xxx",
  "highlight_id": "yyy",
  "timestamp": 45.2,
  "emoji": "🔥",
  "user_id": "anonymous_xxx"
}
```

## 二、非功能需求

### NFR-1：性能
- 首屏加载时间 < 2 秒（3G 网络）
- 视频首帧渲染 < 1 秒
- 互动组件弹出延迟 < 200ms
- 高光点接口响应时间 < 100ms（P99）

### NFR-2：兼容性
- 前端：Android 7.0 (API 24) 及以上
- 服务端：Docker 容器化，支持 amd64/arm64

### NFR-3：可扩展性
- 高光点数据结构预留扩展字段（支持后续 AI 自动标注接入）
- 互动组件采用插件化注册机制，方便新增互动类型

### NFR-4：可用性
- 服务部署在公有云，提供公网 HTTPS 访问
- 核心接口具备降级策略（高光点数据加载失败不影响正常播放）

## 三、用户故事

| ID | 用户故事 | 验收标准 |
|----|----------|----------|
| US-1 | 作为观众，我能浏览短剧列表并选择观看 | 列表展示 ≥ 5 部短剧，点击进入播放页 |
| US-2 | 作为观众，我能控制视频播放（暂停/进度条/全屏） | 播控按钮响应正常，进度条拖拽准确 |
| US-3 | 作为观众，在剧情高光时刻我能一键表达情绪 | 高光点弹出表情组件，点击后有动画反馈 |
| US-4 | 作为观众，我能看到进度条上的情绪热力分布 | 进度条上有颜色热力条，颜色区分情绪 |
| US-5 | 作为观众，我的互动不中断观看体验 | 互动组件不遮挡画面主体，5 秒后自动消失 |

## 四、MVP 闭环流程

```
┌─────────────────────────────────────────────┐
│                  用户端                      │
│  短剧列表 → 点击播放 → 到达高光点           │
│  → 弹出互动组件 → 点击表情 → 动画反馈       │
│  → 进度条热力图更新                          │
└──────────────────┬──────────────────────────┘
                   │ HTTP
┌──────────────────▼──────────────────────────┐
│                 服务端                       │
│  高光点数据 CRUD → 按剧集查询下发            │
│  → 接收互动上报 → 聚合热力数据 → 回传        │
│  → 静态视频资源托管                          │
└─────────────────────────────────────────────┘
```

## 五、数据结构

### 高光点（Highlights）

```typescript
interface Highlight {
  id: string;
  drama_id: string;
  timestamp: number;        // 秒
  type: 'conflict' | 'twist' | 'highlight' | 'romance';
  component: 'emoji_burst';
  emojis: string[];
  label: string;
  duration: number;         // 展示时长，秒
}
```

### 互动记录（Interaction）

```typescript
interface Interaction {
  id: string;
  drama_id: string;
  highlight_id: string;
  user_id: string;
  emoji: string;
  timestamp: number;
  created_at: string;
}
```

### 情绪热力窗口（EmotionWindow）

```typescript
interface EmotionWindow {
  start: number;            // 窗口起始，秒
  end: number;              // 窗口结束，秒
  emotions: Record<string, number>;  // { "🔥": 128, ... }
}
```
