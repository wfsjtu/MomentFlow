# MomentFlow 项目范围说明书

## 项目背景

短剧（微短剧）市场呈爆发式增长——2025 年中国微短剧市场规模达 677.9 亿元，用户规模 6.96 亿。用户对「即时情绪表达」的诉求日益强烈，当前弹幕/评论等方式存在输入门槛高、中断观看体验的问题。**MomentFlow** 旨在为短剧场景打造低门槛、高沉浸的即时互动体验，让用户在看剧过程中「情绪即表达、表达即互动」。

## 创新点（来源于行业调研）

基于对行业前沿的调研，MomentFlow 聚焦以下创新方向：

### 1. 多信号融合高光检测
- 参考 AI 直播高光检测方案（多信号融合：音频峰值 + 画面突变 + 弹幕密度），对短剧内容进行自动高光识别
- 检测维度：音频能量峰值、场景切换频率、角色情绪识别、台词关键词匹配
- 对比传统人工标注，大幅降低内容运营成本

### 2. 一键情绪表达（Emoji Burst）
- 区别于弹幕/评论需要输入文字，提供一键式情绪反馈（预设表情包/Emoji/特效）
- 在高光时刻自动弹出互动组件，用户点击即可表达情绪并看到实时反馈动画
- 降低表达门槛：从「思考→打字→发送」简化为「点击→即时反馈」

### 3. 情绪热力图叠加
- 时间轴上的用户情绪聚合可视化——在进度条上叠加「情绪热力分布」
- 用户可直观看到「哪一段剧情最让观众激动/愤怒/感动」
- 可作为内容推荐和二次传播的数据源

### 4. 智能跳看（Smart Skip）
- 参考爱奇艺「跳看」功能：AI 识别叙事精华点，用户可上下滑动在精彩点间自由跳转
- 实现「千人千面」的内容呈现：同一剧集不同用户可根据兴趣点获得不同观看路径

### 5. Moment Card（高光卡片）
- 将高光片段 + 用户情绪数据生成可分享的「Moment Card」
- 包含：关键帧截图、AI 生成摘要文案、该时刻的情绪统计数据
- 支持分享到社交平台，驱动内容的二次传播和拉新

### 6. 剧情分支互动（Story Branching）
- 在关键剧情节点提供分支选项，用户选择影响后续剧情走向
- 支持实时投票聚合，展示集体选择趋势
- 参考短剧互动手游化趋势（ReelShort、DramaBox 已验证）

### 7. AI 个性化 Recap 生成
- 基于用户观看历史和互动偏好，AI 自动生成个性化精彩集锦
- 支持竖屏/横屏自适应，适配社交平台分发
- 结合 AI 语音合成/字幕生成，降低二次创作门槛

## 项目范围

### 范围内（MVP）

| 模块 | 内容 |
|------|------|
| 短剧列表 & 播放 | 短剧列表页、播放器（暂停/播放/进度条/全屏） |
| 高光点打标 & 下发 | 服务端存储高光点数据，客户端按时间轴接收并触发 |
| 高光剧情互动 | Emoji Burst 一键情绪表达 + 互动特效渲染 |
| 情绪热力图 | 进度条叠加情绪热力分布 |
| 服务部署 | 公有云容器部署，提供公网访问能力 |

### 范围外（后续迭代）

- 剧情分支/续写生成（AI 内容生成）
- 社交共看模式（多用户同步）
- Moment Card 分享
- AI 个性化 Recap
- 用户体系 & 历史记录

## 技术方案概要

### 混合架构：Node.js + Python 双后端协同

**设计理念：** 主业务用 Node.js（实时性、高并发），AI 分析用 Python（生态最强），两者通过 HTTP REST/gRPC 通信。

| 层级 | 技术选型 |
|------|----------|
| 前端（Android） | Kotlin + Jetpack Compose + ExoPlayer |
| **主业务服务** | **Node.js + NestJS** |
| **AI 分析服务** | **Python + FastAPI** |
| 数据库 | PostgreSQL + Redis（缓存高光点数据） |
| 存储 | OSS/COS（短剧视频文件） |
| 部署 | Docker Compose + 公有云容器服务 |
| AI 框架 | PyTorch / HuggingFace Transformers（多信号融合高光检测） |
| 视频/音频处理 | FFmpeg + OpenCV + Librosa（Python 侧） |

```
└──────────────────────────┬──────────────────────────────────┘
                           │ HTTP/WebSocket
┌──────────────────────────▼──────────────────────────────────┐
│           主业务服务 (Node.js + NestJS + PostgreSQL + Redis) │
│  - 短剧列表 / 播放管理                                      │
│  - 高光点数据下发（缓存 Redis）                              │
│  - 互动数据上报 / 聚合（情绪热力图）                          │
│  - WebSocket 实时同步（可选）                                │
│  ┌────────────────────────────────────────────────────────┐ │
│  │  AI 服务调用接口 (HTTP 客户端)                          │ │
│  └────────────────────┬───────────────────────────────────┘ │
└───────────────────────┼───────────────────────────────────────┘
                        │
┌───────────────────────▼───────────────────────────────────────┐
│           AI 分析服务 (Python + FastAPI + PyTorch/HuggingFace) │
│  - 多信号融合高光检测（音频峰值 + 画面突变 + 台词关键词）       │
│  - 场景分类（冲突/反转/名场面/撒糖）                          │
│  - 批量剧集分析任务队列                                      │
└───────────────────────────────────────────────────────────────┘
```

### 服务间通信

- **协议：** HTTP REST（MVP 阶段优先），后续可升级为 gRPC（高性能）
- **数据格式：** JSON
- **示例：** Node.js 调用 Python AI 服务进行视频分析
